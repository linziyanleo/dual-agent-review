# Plan v2: 给 /api/login 加 token-bucket rate limit

## Context / Goals
- 当前 `/api/login` 无任何 rate limit，被 brute-force 撞过几次（incident #INC-412）。
- 目标：把对单个 IP 的登录尝试限制到 **每分钟 10 次**；超限返回 `429 Too Many Requests`。
- 不影响正常用户体验（人类 1 分钟内不会试 10 次密码）。

## Non-goals
- 不做 CAPTCHA、不接 WAF —— 那是另一张工单。
- 不做按用户名 rate limit（先按 IP，下个迭代再加 username 维度）。
- 不做分布式 rate limit（当前后端只跑单实例）。

## Trusted proxy 前置条件（v2 新增，回应 F-2）
- `c.ClientIP()` 在 gin 默认信任 `X-Forwarded-For`；公网部署必须 explicit trust 列表。
- 在 `cmd/server/main.go` 调用 `engine.SetTrustedProxies([]string{"10.0.0.0/8"})`（替换为实际反代 IP 段）。
- 没这一步，rate limit 形同虚设。

## Proposed approach
1. 在 `internal/middleware/ratelimit.go` 用 `golang.org/x/time/rate.Limiter`，每 IP 一个 limiter (`rate.Every(6*time.Second), burst=10`)。
2. 用 `sync.Map[string]*entry` 存 limiters；每条 entry 含 `*rate.Limiter` + `lastSeen time.Time`。
3. 起一个 sweep goroutine（30 min tick），删除 `time.Since(lastSeen) > 30*time.Minute` 的 entry。
4. 把 limiter 挂到 `/api/login` route：`router.POST("/api/login", ratelimit.ByIP(), handler.Login)`。
5. 超限分支：return `c.JSON(429, gin.H{"error": "too many login attempts"})`。

## Affected files
- `internal/middleware/ratelimit.go`（新建）
- `internal/middleware/ratelimit_test.go`（新建）
- `cmd/server/routes.go`（挂中间件）
- `cmd/server/main.go`（`SetTrustedProxies`，v2 新增，回应 F-2）

## Risks & open questions
- 进程重启 token 计数清零 —— 攻击者可以触发崩溃 + 重启绕过。先记 risk，不处理。
- sweep goroutine 在 SIGTERM 时无优雅退出；目前 process 直接死，可接受。
- IP 取值依赖 `SetTrustedProxies`；未配置就上线 = 没装 limit，部署 checklist 加一条。

- 写 `ratelimit_test.go`：等 6.1 秒后能续 1 次（10 tokens/min = 1 token / 6s）。
